    \section{Arguments and parameters}
    A value that can be used as an argument to a function is a \textit{first-class} value.

    Haskell is \textit{lexically scoped}.

    Incomplete pattern matches applied to data they don't handle will return \textit{bottom}, a non-value used
    to denote that the program cannot return a value or result.
    
    \section{Pattern matching against data constructors}
    Some data constructors have parameters, and pattern matching can let us expose
    and make use of the data in their arguments.


    \begin{Def}[newtype]
    newtype is a special case of \textbf{data} declaration. It permits only 1 constructor and
    only one field.
    \end{Def}
    \section{Higher-order function}
    \begin{Def}[Higher-order function]
        \textit{Higher-order functions} are functions that accept functions as arguments. Functions are 
        values.
    \end{Def}
    
    Guards always evaluate sequentially, so it should be ordered from the case that is most
    restricive to the one that is least restrictive.

    \subsection*{Function composition and "pointfree" style}

    \begin{Def}[Currying]
        \textit{Currying} is the process of transforming a function that takes multiple arguments into
        a series of functions which each take 1 argument and return one result.
    \end{Def}

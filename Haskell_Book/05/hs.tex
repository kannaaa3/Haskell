    \section{Partial application}
    
    The word "\textit{sectioning}" refers to partial application of infix operators.

    \begin{Def}[Parametricity]
        \textit{Parametricity} means that the behavior of a function with respect to the types of its (parametrically polymorphic) arguments is uniform. The behavior cannot change just because it was applied to
        an argument of a different type.
    \end{Def}

    \section{Type inference}
    Haskell does not obligate us to assert a type for every expression or value because it has \textit{type inference}.
    \begin{Def}[Type inference]
        Type inference is an algorithm for determining the types of expressions. Haskell will
        infer the most generally applicable (polymorphic) type that is still correct.
    \end{Def}

    \section{Determine the type}
    \begin{Def}[Monomorphism restriction]
        Top-level declarations by default will have a concrete type if any can be determined.

    \end{Def}

    \section{Definition}
    \begin{enumerate}
        \item \textbf{Polymorphism}: in Haskell this will usually manifest as \textit{parametric} or \textit{ad-hoc polymorphism}.
        \item \textbf{Type inference:} Infer principal types from terms without needing explicit type
            annotations.
        \item \textbf{Type variable} is a way to refer to an unspecified type or set of types in Haskell type
            signatures.
        \item \textbf{Typeclass:} a means of expressing faculties or interfaces that multiple datatypes may have
            in common.
        \item \textbf{Ad-hoc polymorphism} (\textit{constrained polymorphism})
        \item \textbf{Module}: The unit of organization
    \end{enumerate}
